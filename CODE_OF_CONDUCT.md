# Code Of Conduct

CommandGuard should be useful to beginners, researchers, and maintainers. Keep discussions practical, respectful, and safe.

## Expected Behavior

- Be kind to new Linux users.
- Explain security concepts without shaming people.
- Keep feedback specific and actionable.
- Use inert examples when discussing harmful command patterns.
- Respect maintainers' time and scope decisions.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Publishing live malware or real stolen data.
- Encouraging users to run unsafe commands.
- Dismissing beginner questions as stupid.

## Enforcement

Maintainers may edit, hide, or remove comments and contributions that make the project unsafe or hostile.
