# Security Policy

CommandGuard is a security tool, but it is also an early prototype. Please treat it as a research and defense project, not as a complete endpoint protection product.

## Supported Versions

| Version | Status |
| --- | --- |
| 0.1.x | Prototype, best-effort security fixes |

## Reporting Security Issues

If you find a vulnerability in CommandGuard itself, please report it privately if possible.

Until a public contact address exists, open a GitHub security advisory or create a minimal issue that says you have a private security report without sharing exploit details.

Useful details:

- affected version or commit;
- operating system and terminal emulator;
- exact command or paste sequence if safe to share;
- expected behavior;
- actual behavior;
- whether the issue causes bypass, unsafe execution, crash, or data exposure.

## Do Not Submit

Please do not attach:

- live malware;
- real stolen credentials;
- working command-and-control endpoints;
- exploit chains against third-party systems.

Use inert payloads and reserved domains such as `example.test`.
